Compiling Mapper from Source


Source is available if you want to modify the program or run it on a Linux system.

Prerequisites

Mapper uses QT as a programming framework. QT can be downloaded from QT download. QT uses SQLite3 as a database engine. However, the version of SQLite3 that is included in QT does not allow for the loading of user defined functions that the SQL queries used in Mapper need. For this reason, you must make sure that SQLite3 is compiled with the SQLITE_ENABLE_LOAD_EXTENSION defined. .   If you already have SQLite on your PC, QT can be configured to use the version on your computer.In either case, you will have to also download the QT source and compile the Sqlite SQL plugin. . SQLite3 source can be downloaded from  the SQLite Download Page. If you wish to use Sql Server or MySql databases, you will also have to compile QT with plugins for those database types. My recommendation is to download the SQLite3 source and compile it. Then, recompile the QT SQLITE SQL plugin to have QT use the "system: sqlite library or dll instead of an internalone.

Mapper was originally developed using version 4.8 of QT. However there are newer versions of QT that have some enhancements that are desirable. At the present time< September 2016, QT version 5.6.1 seems to work the best. QT as of version 5.7 no longer supports QWebkit and instead offers QWebEngine as an alternative. Mapper compiled on QT versions later than 5.4 will use the QWebEngine module instead of QWebKit. QWebkit and QWebEngine provide the ability of Mapper to display a map using Google Maps in a web widget.

Windows users will have to also install Microsoft Visual Studio  on their computers. Note that there are two varieties of QT for Windows, one requiring Visual Studio and the other MinGW. I used Visual Studio 2008 for QT 4.8 but newer versions of Visual Studio(2010 & 2012) are available. Also when downloading QT, look for the "Archive for old versions" link which will let you download either a QT version 4 or 5. The version of Microsoft Visual studio that you will need will in turn depend on which version of QT you are using. For example, QT 5.6.1 comes compiled with MSVC 2013 or MSVC 2015 or MinGw while QT version 5.4 comes compiled with MSVC 2010 or MSVC 2013 or MinGw

Another prerequisite is the Fossil program which can be downloaded from the Fossil download page. Fossil is needed to download the source from my server. Fossil will run on Windows or Linux. Once you have installed Fossil on your computer, and have inserted it's location in the PATH, follow these steps to download the source.
  1. Select a root directory where the source will reside. In my case, I have a directory called 'Projects' although it can be any name you choose.
  2. In the 'Projects' directory, create a directory named 'mapper_QT'. Again, the name you choose for this directory is not important.
  3. Change to the 'mapper_QT' directory.
  4. Clone the source repository with the command: 'fossil clone http://acksoft.dyndns.biz:1080/cgi-bin/fossil/mapper_QT mapper_qt.fossil'. This will create a file named 'mapper_qt.fossil' which is your local repository copy.
  5. Next, issue the command 'fossil open mapper_qt.fossil'. This will download the source and other files in the project. It will create a directory 'mapper in the '../mapper_qt/' directory containing directories: 'Console', 'functions', 'mapper_app', 'Resources', 'sqlfun' and 'wiki'.
  6. In QtCreator, open the 'mapper.pro' project file.
  7. The latest source from the fossil repository can be updated at any time by running the command fossil update from the command line.
Mapper source can also be downloaded from GitHub: https://github.com/allenck/mapper
